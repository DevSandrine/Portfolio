Stage 1

Stage chez Xpilab : agence digitale

(entreprise individuelle) J'ai dans le cadre de ma formation effectué deux stages. Ces 2 stages ont été réalisé au même endroit,chez Xpilab situé à Brunoy , dans une agence digitale dont les activités peincipale étaient, le développement web, Stratégie marketing & Communication digitale , création de contenu our les petites et moyennes entreprises (PME) françaises, ainsi que les associations désirant faire la transition numérique.
  • Mon projet lors de ce premier stage était de développer un site vitrine responsive : cad qui consiste à rendre un site web accessible et adaptable à tous les devices : tablettes, smartphones, ... pour cela j'ai utilisé le langage html/css puis j'ai dû faire de la veille sur les différentes technologies à utilisées, notemment: bootstrap(est un framework CSS qui organise et gère la mise en page d'un site web) , les media queries(il est utilisé pour modifier l'affichage d'une page web selon la hauteur et la largeur de l'écran utilisé )et les breakpoint: (Un breakpoint (ou point de rupture) se réfère à la taille imposée par la fenêtre de rendu du navigateur qui fera varier proportionnellement le rendu visuel du site web affiché. Par exemple sur un PC moderne, vous aurez une zone d affichage du navigateur de 1 400 pixels de large.). J'ai dû cherché une fonctionnalité, qui devait rendre plus agréable la présentation des sites demos aux client. Pour cela j'ai choisi d'intégrer une fonctionnalité qui permet de scroller le site demo par le client.


    • J'ai commencé mon stage, en faisant d'abord de la veille sur Gitlab:
  • Fonctionnement de base de Git : Comprendre les concepts fondamentaux de Git, tels que les dépôts (repositories), les commits, les branches, les fusions (merges), etc.
  • Installation et configuration de Git : Apprendre à installer Git sur votre système et à le configurer selon vos besoins, notamment en configurant votre nom d'utilisateur, votre adresse e-mail, etc.
  • Utilisation de Git en ligne de commande : Se familiariser avec les commandes Git les plus couramment utilisées, telles que git init, git clone, git add, git commit, git push, git pull, git merge, etc.
  • Gestion des branches : Comprendre l'importance des branches dans Git et comment les utiliser efficacement pour travailler sur des fonctionnalités isolées, des correctifs de bogues, etc.

  • J'ai également utilisé la plateforme Notion , c'est un espace de travail , une plateforme qui permet d'organiser les tâches à faire au quotidien pour les différents membres d'une équipe, c'est comme une sorte de checklist des tâches de tous les jours,et ça permet aussi de partager des notes et tout type de documents.(un carte par membre, ils retrouvent les tâches à faires de chaque jour et check les tâches réaliser au fur et à mesure de l'accomplissement).Permet d'avoir une meilleur organisation et gestion du temps.
